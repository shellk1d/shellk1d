Forest

HackTheBox Windows Easy Active Directory Kerberoasting AS-REP Roasting BloodHound 10 Nov 2024

Introducción

Forest es una máquina Windows de dificultad fácil de la plataforma HackTheBox. Se centra principalmente en la enumeración de Active Directory (AD) y en la explotación de vectores clásicos como AS-REP Roasting y escalada de privilegios abusando de permisos de replicación de dominio (DCSync).

01. Enumeración Inicial

Comenzamos realizando un escaneo exhaustivo de puertos utilizando la herramienta nmap:

nmap -sTCV -p- --min-rate 5000 10.10.10.161

La salida nos muestra varios puertos abiertos típicos de un Controlador de Dominio (DC) de Active Directory:

  • Puerto 88: Servicio Kerberos.
  • Puerto 135/445: Servicios MSRPC y SMB.
  • Puerto 389/3268: Protocolo LDAP / Global Catalog.

02. Acceso Inicial (AS-REP Roasting)

Dado que no tenemos credenciales, intentamos enumerar los usuarios del dominio explotando sesiones anónimas en LDAP o SMB. Una vez generada la lista de usuarios válidos, probamos la técnica de AS-REP Roasting.

Esta técnica se aprovecha de las cuentas de usuario que tienen deshabilitada la preautenticación de Kerberos (DONT_REQ_PREAUTH).

Utilizamos la suite GetNPUsers.py de Impacket para solicitar los tickets TGT sin necesidad de contraseña:

GetNPUsers.py htb.local/ -usersfile users.txt -format hashcat -outputfile hashes.txt

El script extrae con éxito el hash de la cuenta svc-alfresco:

$krb5asrep$23$svc-alfresco@HTB.LOCAL:12a67e416d86...

Procedemos a crackear el hash localmente con hashcat y el diccionario rockyou.txt:

hashcat -m 18200 hashes.txt /usr/share/wordlists/rockyou.txt

La contraseña se revela como: s3cr3tPassword

03. Movimiento Lateral y Escalada de Privilegios

Iniciamos sesión en el sistema usando Evil-WinRM con las credenciales de svc-alfresco:

evil-winrm -i 10.10.10.161 -u svc-alfresco -p s3cr3tPassword

Una vez dentro, ejecutamos SharpHound para recolectar datos de Active Directory y analizarlos visualmente con BloodHound:

  1. Subimos el recolector SharpHound.ps1.
  2. Ejecutamos Invoke-BloodHound -CollectionMethod All.
  3. Descargamos el archivo zip resultante y lo importamos en BloodHound.

Al analizar las relaciones de pertenencia a grupos en BloodHound, encontramos que el usuario svc-alfresco es miembro del grupo Service Accounts, el cual a su vez pertenece a otros grupos de administración delegada que nos permiten agregar usuarios al grupo Exchange Windows Permissions.

04. Compromiso de Dominio (DCSync)

Al añadir nuestra sesión o cuenta comprometida a ciertos grupos delegados de Exchange, obtenemos permisos implícitos para realizar sincronizaciones de dominio (WriteDacl sobre el objeto del dominio).

Esto nos permite ejecutar un ataque DCSync simulando un Controlador de Dominio secundario que solicita réplicas de contraseñas.

Ejecutamos secretsdump.py para dumpear los hashes del dominio entero:

secretsdump.py htb.local/svc-alfresco:s3cr3tPassword@10.10.10.161 -just-dc-user Administrator

¡Obtenemos con éxito el hash NTLM del usuario Administrador del dominio!

Administrator:500:aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0:::

Finalmente, realizamos un ataque Pass-the-Hash con evil-winrm usando el hash NTLM para obtener acceso administrativo completo:

evil-winrm -i 10.10.10.161 -u Administrator -H 31d6cfe0d16ae931b73c59d7e0c089c0

¡Máquina Forest comprometida por completo!
